# 刷题入口与左右分栏真格子纸设计稿

## 1. 背景与目标
- 背景：当前首页“刷题”按钮尚未接入真实流程，`/practice` 页面仍是占位；后端已具备 `POST /api/practice/submit` 自评提交能力。
- 目标：上线“先筛选再开始”的单题刷题闭环，并提供左题右写作格子纸界面，满足主观题作答习惯。
- 非目标（第一版不做）：
  - 倒计时
  - 多题连续会话
  - 云端草稿同步
  - 复杂抽题优先级（未练优先/久未练优先）

## 2. 已确认决策
1. 点击“刷题”后先进入筛选面板，不直接开题。
2. 筛选项第一版仅保留：`题库`、`题型`。
3. 开始后是“单题即止”流程（做完一题后返回筛选）。
4. 参考答案为“先隐藏，手动点击显示后再自评”。
5. 自评分采用 `0-5`（与现有后端一致）。
6. 提交成功后返回筛选页，并保留上次筛选条件。
7. 采用自动单用户模式（本地单人使用，无需登录）。
8. 右侧输入区为真格子纸：每字符占 1 格。
9. 格子规格为每页 `24 行 * 25 格`，可翻页新增。
10. 当前页写满后自动新增下一页并继续输入。
11. 支持整段粘贴并按格连续填充。
12. 草稿按题目 ID 本地自动暂存并可恢复。

## 3. 页面与交互设计
### 3.1 页面结构
- 路由：`/practice`
- 视图状态：
  - `filter`：筛选态（题库、题型、开始刷题）
  - `solving`：作答态（左题右纸）
  - `submitting`：提交中（按钮禁用）

### 3.2 filter 态
1. 页面顶部显示“刷题”标题和简短提示。
2. 渲染题库下拉、题型下拉（题型可空表示不限）。
3. 点击“开始刷题”触发 `GET /api/practice/next`。
4. 无题时展示“当前筛选下暂无题目”，停留在筛选态。

### 3.3 solving 态（核心）
- 左侧题目区：
  - 题目摘要、材料、问题、分值、字数要求、题型
  - “显示参考答案”按钮（点击后展开参考答案）
  - 自评分 0-5 单选/按钮组
  - “提交自评并完成本题”按钮
- 右侧格子纸区：
  - 真格子输入：24 行 × 25 格
  - 分页：上一页/下一页/新增页 + `第 n 页`
  - 自动分页：当前页写满 600 字后自动到新页
  - 独立滚动：右侧滚动不影响左侧题目区

### 3.4 提交流程
1. 用户可先看/不看参考答案。
2. 选择 0-5 自评分并提交。
3. 调用 `POST /api/practice/submit` 成功后：
  - 中央短提示“已保存”
  - 清除当前题目草稿
  - 回到 `filter` 态并保留筛选条件

## 4. 格子纸编辑器细则
### 4.1 数据模型
- `pages: string[]`：每项为该页文本。
- 每页容量 `PAGE_CAPACITY = 24 * 25 = 600`。
- 展示时将页文本映射为 600 个格子（不足补空）。

### 4.2 输入规则
- 每字符占 1 格（中文、英文、数字、符号统一）。
- 输入从当前光标格写入，光标后移。
- `Backspace` 删除前一格；`Delete` 删除当前格。
- 方向键按格移动，跨行/跨页遵循线性索引。

### 4.3 粘贴规则
- 支持整段粘贴。
- 粘贴内容预处理：去除不可见控制字符，保留换行并转为普通字符流。
- 从当前格开始连续填充，超出当前页自动创建后续页。

### 4.4 分页与草稿
- 手动分页：上一页、下一页、新增页。
- 自动分页：写满即建新页并聚焦新页首格。
- 暂存键：`draft:{questionId}`。
- 保存节流：约 500ms。
- 提交成功时清理对应草稿。

## 5. 后端接口与数据流
### 5.1 自动单用户
- 新增服务：`resolveLocalUser()`。
- 行为：按固定邮箱（默认 `local@tuanyuan.app`）查找；不存在则创建。
- 目的：前端不处理登录和 userId 输入。

### 5.2 新接口：`GET /api/practice/next`
- 入参：
  - `bankId`（必填）
  - `type`（选填）
- 流程：
  1. `resolveLocalUser()` 获取 `userId`
  2. 按筛选查题（含 `materials`）
  3. 随机返回 1 题
- 出参：`{ userId, question }`
- 错误：
  - 缺少 `bankId` -> 400
  - 无可用题 -> 404

### 5.3 既有提交接口复用
- `POST /api/practice/submit`
- 请求体：`{ userId, questionId, selfScore, spentSec? }`
- 保持现有评分合法性与排期逻辑。

## 6. 错误处理
- next 失败：在筛选态展示明确错误文案。
- submit 失败：保留作答内容，不清草稿，不切回筛选态。
- 本地存储异常（极少）：降级为仅内存态并提示“草稿未写入本地”。

## 7. 测试与验收标准
1. 首页“刷题”可进入 `/practice`。
2. `/practice` 可按题库/题型取到 1 题；空结果有提示。
3. 左右分栏渲染正确，右侧可输入真格子。
4. 格子纸支持整段粘贴、自动分页、手动翻页。
5. 草稿可恢复，提交成功后草稿被清理。
6. 提交后出现短提示并返回筛选态，筛选值仍保留。
7. 现有题库管理、导入与回归测试不受影响。
