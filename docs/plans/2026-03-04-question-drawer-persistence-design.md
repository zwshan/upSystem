# 新建题目与抽屉编辑真实落库设计文档

## 1. 目标
- 将 `/banks` 页面当前“前端壳”升级为真实可落库流程。
- 覆盖两个核心动作：
  - 新建题目（create）
  - 抽屉编辑（edit）
- 统一约束：沿用当前 Dark 毛玻璃视觉，不引入第二套风格。

## 2. 已确认决策
- 新建与编辑共用一套表单组件。
- 新建与编辑都使用右侧抽屉作为交互容器。
- 保存后抽屉不自动关闭，提示成功并允许继续编辑。
- 材料输入使用单一美观大文本框，不做分段增删 UI。
- 材料分段规则：按空行切分（保存时自动转换为 `materials[]`）。
- 抽屉存在未保存改动时，关闭需二次确认。

## 3. 组件与状态设计

### 3.1 组件结构
- `BanksPage`
  - 负责列表数据拉取、筛选与行操作。
  - 控制抽屉开关及 `mode=create/edit`。
- `QuestionDrawer`
  - 只负责容器、标题、关闭逻辑与确认。
- `QuestionForm`
  - 负责字段渲染、校验、材料转换、提交动作。

### 3.2 抽屉模式
- `create`：
  - 空白初始值。
  - 提交到 `POST /api/questions`。
- `edit`：
  - 打开后先 `GET /api/questions/:id` 回填。
  - 提交到 `PATCH /api/questions/:id`。

### 3.3 状态机
- `loading`（仅 edit 初次回填）
- `editing`
- `submitting`
- `success/error`
- `dirty` 标记（用于关闭拦截）

## 4. 表单字段与校验
- 必填：`summary`、`prompt`、`scoreValue`、`wordLimit`、`questionType`、`materialText`
- 可选：`referenceAnswer`、`tags`
- 题型来源：`GET /api/question-types`（内置 + 自定义）
- 自定义题型：`POST /api/question-types`

## 5. 材料转换规则
- 输入预处理：`\r\n` -> `\n`
- 以“一个或多个空行”分割段落
- 每段 `trim`，过滤空段
- 转换后输出：
  - `[{ seq: 1, content: "..." }, ...]`
- 无有效段时阻断提交并提示“题目材料不能为空”

## 6. 数据流与接口联动

### 6.1 新建（Create）
1. 填表 -> 前端校验 + 材料切分。
2. 调 `POST /api/questions`。
3. 成功后：
   - 列表插入新题或重拉当前页。
   - 抽屉保留打开。
   - `dirty=false`，提示“已保存，可继续编辑”。

### 6.2 编辑（Edit）
1. 点击行编辑 -> 打开抽屉 -> `GET /api/questions/:id`。
2. 回填到表单（`materials[]` 合并回 `materialText`，段间空行）。
3. 保存时重新切分并 `PATCH /api/questions/:id`。
4. 成功后局部更新列表行展示字段。

### 6.3 关闭保护
- `dirty=true` 且点击关闭：
  - 弹确认：“有未保存更改，是否放弃？”
  - 确认放弃才关闭。

## 7. 异常处理
- 400：字段级提示（校验错误映射到对应输入项）。
- 404（编辑对象不存在）：提示后关闭抽屉并刷新列表。
- 500：保留当前表单输入，不清空。
- 网络失败：显示重试提示，禁止静默失败。

## 8. 验收标准
- 新建流程真实写库成功，列表可见新题。
- 编辑流程真实更新数据库，列表与详情一致。
- 材料按空行切分稳定且可回显。
- 未保存关闭有二次确认。
- `/banks` 页面视觉风格与首页保持一致。
