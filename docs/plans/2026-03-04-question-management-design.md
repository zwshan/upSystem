# 题目管理模块设计文档

## 1. 目标与定位
- 模块目标：为“团圆学习平台”提供可持续的题目资产管理能力，支撑前期高频导入、后期低频手动维护。
- 使用阶段特征：
  - 初期：批量 JSON 导入后做局部修订。
  - 后期：少量手动录题与维护。
- 风格约束：题目管理页必须与首页保持统一的 Dark 毛玻璃风格，不允许独立主题分裂。

## 2. 已确认需求
- 工作流优先级：导入后修订优先，其次手动录题。
- 导入后修订能力：既要逐题编辑，也要批量编辑；逐题编辑优先打磨。
- 删除策略：永久删除（无回收站）。
- 导入策略：字段错误题跳过，其他题继续导入，输出错误报告。
- 题型策略：内置枚举 + 允许新增自定义类型（可复用）。
- 重复题策略：默认直接导入为新题，不做阻断。

## 3. 信息架构（列表优先）

### 3.1 页面入口
- 从右上角 `题库管理` 进入。
- 默认落在题目列表主视图。

### 3.2 页面布局
- 顶部：题库切换 + 搜索 + 筛选（题型、分值区间、字数要求、标签）。
- 主体：题目列表（摘要、题型、分值、材料段数、更新时间）。
- 侧边：抽屉式逐题编辑器（点击列表项打开）。
- 工具栏：`新建题目`、`导入 JSON`、批量操作。

### 3.3 视觉一致性要求
- 统一使用现有设计 token：背景、玻璃、边框、阴影、文字层级。
- 表格、抽屉、弹窗、输入框遵循同一主题变量。
- 交互反馈（hover/focus/disabled/toast）与首页手感一致。

## 4. 核心交互流程

### 4.1 JSON 导入流程（默认 skip invalid）
1. 点击 `导入 JSON`。
2. 上传/粘贴 JSON，选择目标题库。
3. 执行预检，展示：总数、可导入数、错误数、错误明细。
4. 执行导入（默认 `skipInvalid=true`）。
5. 导入报告页显示成功/失败统计与错误详情，并提供“仅看新导入题目”筛选。

### 4.2 逐题编辑流程
1. 点击列表行打开右侧抽屉。
2. 编辑字段：摘要、问题、材料（多段增删排序）、分值、字数要求、题型、标签、参考答案。
3. 点击保存后局部刷新当前行。
4. 支持“上一题/下一题”连续修订。

### 4.3 批量编辑流程
1. 多选题目，启用批量工具栏。
2. 可执行：批量改题型、批量改标签、批量改分值、批量删除。
3. 执行前显示“影响条数”预览并二次确认。
4. 执行后展示成功/失败统计。

### 4.4 删除流程（永久）
- 单题删除：抽屉内危险操作 + 二次确认。
- 批量删除：工具栏危险操作 + 输入确认词（例如 `DELETE`）。
- 删除后即时从列表消失，不进入回收站。

### 4.5 题型管理流程
- 题型选择器默认展示内置项。
- 支持“新增自定义类型”，保存后进入统一字典，可在筛选和录题复用。

## 5. 数据与接口设计

### 5.1 数据扩展
- 新增表：`question_type_dict(id, name, is_builtin, created_at)`。
- 保留既有表：`question_bank`、`question`、`question_material`。

### 5.2 API 设计（在现有基础上扩展）
- 保留现有：
  - `GET/POST /api/banks`
  - `POST /api/questions`
  - `POST /api/questions/import`
- 新增：
  - `GET /api/questions`（分页、搜索、筛选）
  - `GET /api/questions/:id`
  - `PATCH /api/questions/:id`
  - `DELETE /api/questions/:id`
  - `POST /api/questions/batch-update`
  - `POST /api/questions/batch-delete`
  - `GET/POST /api/question-types`

### 5.3 导入接口增强
- 新增参数：`dryRun`（仅预检不写库）。
- 返回结构增强：
  - `importedCount`、`skippedCount`、`errorCount`
  - `errors[]`（含 index/field/message）
  - `importedIds[]`

## 6. 状态与数据流
- 首屏加载：题库列表、题型字典、第一页题目。
- 筛选变化：仅刷新列表，尽量保留抽屉上下文。
- 保存成功：局部更新列表行和抽屉数据。
- 批量操作成功：刷新当前筛选集与统计。

## 7. 异常与保护机制
- JSON 非法：直接阻断并提示解析错误。
- 字段级不合法：记入错误清单，不阻断合法题导入。
- 批量部分失败：返回明细，不做整批回滚。
- 编辑目标不存在：提示并关闭抽屉，返回列表。
- 危险操作统一双确认。
- 抽屉存在未保存改动时关闭需要确认。

## 8. 验收标准（题目管理专项）
- 可在 10 分钟内完成：
  - 导入 100 题
  - 修订 10 题
  - 批量改 20 题标签/题型
  - 删除 3 题
- 导入错误项可追踪到具体题号和字段。
- 列表/抽屉/弹窗主题风格与首页一致，无割裂视觉。
- 对应 API 和页面流程具备自动化测试覆盖。
